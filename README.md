# regalia

An asynchronous IPv4 scanner written in Rust that finds public Minecraft servers (Java Edition) and logs their status to SQLite.

## How it works

Address order isn't sequential — it's generated by an LCG based on the BlackRock algorithm (the one used in masscan/zmap), which lets the scanner walk the entire address space without repeats and without keeping explicit traversal state.

For each generated address:

1. Private, loopback, multicast, link-local, and other reserved ranges are filtered out (RFC 1918, RFC 5737, CGNAT, etc.)
2. A TCP connection is opened on port `25565` with a 3-second timeout
3. If the connection succeeds, a Server List Ping is sent (via `mc-server-status`), also with a 3-second timeout
4. If the server responds, the data (IP, port, MOTD, online/max players, version, protocol) is written to `regalia.db`

Concurrency is capped by a semaphore at 5000 simultaneous checks. Each server's history is stored as individual snapshots (`snapshots` table) rather than just the latest state, so online counts can be tracked over time.

## Running

```bash
cargo run --release
```

On first run, `regalia.db` is created and migrations from `./migrations` are applied. Progress is logged every million addresses checked.

Logging goes through `tracing` — level is set the usual way via `RUST_LOG`:

```bash
RUST_LOG=info cargo run --release
```

## Stack

- `tokio` — async runtime
- `sqlx` (SQLite) — result storage, compile-time query checking
- `mc-server-status` — Server List Ping protocol
- `tracing` — logging

## Status

A full IPv4 sweep takes a while even at 5000 concurrent connections — this is an I/O-bound scanner, not a fast one. Currently sitting at 1.5M+ snapshots collected.

## Notes

- Java Edition only (Bedrock isn't supported)
- Port is hardcoded to `25565` — servers on non-standard ports won't be found
- No rate/bandwidth limiting beyond the semaphore — keep that in mind if running on rented servers or in the cloud
